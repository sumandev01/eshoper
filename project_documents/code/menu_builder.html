Menu Builder Architecture

Technical insights into the dynamic Menu Builder system implemented in the E-Shopper CMS.

1. Database Schema

The Menu Builder is built on a two-table relationship structure.

menus Table

Stores the structural menu blocks/locations.

  • id (PK)
  • name: Display name in the admin panel (e.g., "Main Header Menu").
  • location: The unique string identifier used to fetch the menu in Blade templates (e.g., header_main, footer_company).
  • status: Boolean to toggle visibility.

menu_items Table

Stores the actual links inside a menu.

  • id (PK)
  • menu_id: FK to menus.id (Cascades on delete).
  • title: Display text of the anchor tag.
  • type: Enum/String defining the resolution strategy (system, page, category, custom).
  • reference_id: Contextual data based on type (route name, page id, category id).
  • url: Stores the URL string only if type=custom.
  • order: Integer for sorting items ascending.

2. MVC Implementation

Models

  • App\Models\Menu (hasMany items)
  • App\Models\MenuItem (belongsTo menu)

Controller

App\Http\Controllers\Admin\MenuController

  • Handles index (viewing the split layout).
  • Handles store (creating new menu locations).
  • Handles storeItem (adding links based on the 4 form types).
  • Handles destroyItem (removing a link).

3. Frontend Rendering Strategy

All hardcoded static links have been removed from the Blade templates. To render a menu, we query the Menu model by its location column. Below is an example of the Header Main menu.

@php
    $headerMenu = \App\Models\Menu::with(['items' => function($q) { 
        $q->orderBy('order'); 
    }])->where('location', 'header_main')->first();
@endphp

@if($headerMenu)
    @foreach($headerMenu->items as $item)
        @php
            $linkUrl = '#';
            if ($item->type == 'custom') {
                $linkUrl = $item->url;
            } elseif ($item->type == 'system') {
                $linkUrl = $item->reference_id == 'root' ? route('root') : route($item->reference_id);
            } elseif ($item->type == 'page') {
                $page = \App\Models\Page::find($item->reference_id);
                $linkUrl = $page ? route('page', $page->slug) : '#';
            } elseif ($item->type == 'category') {
                $cat = \App\Models\Category::find($item->reference_id);
                $linkUrl = $cat ? route('categoryProducts', $cat->slug) : '#';
            }
        @endphp
        <a href="{{ $linkUrl }}">{{ $item->title }}</a>
    @endforeach
@endif

4. Default Seeding

The default menus and their initial structure are populated via database/seeders/MenuSeeder.php. This ensures that when a new user installs the project, the header and footer aren't empty and the basic structure (Home, Products, About) is pre-configured.
