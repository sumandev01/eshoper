Developer Documentation - Menu Builder Architecture

This document provides technical insights into the Menu Builder system implemented in the eShopper CMS.

Database Schema

The Menu Builder is built on a two-table relationship: menus and menu_items.

1. menus Table

Stores the structural menu blocks/locations.

  • id (PK)
  • name: Display name in the admin panel (e.g., "Main Header Menu").
  • location: The unique string identifier used to fetch the menu in Blade templates (e.g., header_main, footer_company).
  • status: Boolean to toggle visibility.

2. menu_items Table

Stores the actual links inside a menu.

  • id (PK)
  • menu_id: FK to menus.id (Cascades on delete).
  • title: Display text of the anchor tag.
  • type: Enum/String defining the resolution strategy.
    • system: Hardcoded application routes (e.g., Home, Products).
    • page: Dynamically generated CMS pages.
    • category: Product categories.
    • custom: Direct URL strings.
  • reference_id: Contextual data based on type.
    • If type=system, stores the route name (e.g., root, about).
    • If type=page, stores Page::id.
    • If type=category, stores Category::id.
  • url: Stores the URL string only if type=custom.
  • order: Integer for sorting items ascending.

MVC Implementation

  • Models:
    • App\Models\Menu (hasMany items)
    • App\Models\MenuItem (belongsTo menu)
  • Controller: App\Http\Controllers\Admin\MenuController
    • Handles index (viewing the split layout).
    • Handles store (creating new menu locations).
    • Handles storeItem (adding links based on the 4 form types).
    • Handles destroyItem (removing a link).

Frontend Rendering Strategy

All hardcoded static links have been removed from header.blade.php and footer.blade.php. To render a menu, we query the Menu model by its location column.

Example Rendering Loop (Header Main):

@php
    $headerMenu = \App\Models\Menu::with(['items' => function($q) { $q->orderBy('order'); }])->where('location', 'header_main')->first();
@endphp

@if($headerMenu)
    @foreach($headerMenu->items as $item)
        @php
            $linkUrl = '#';
            if ($item->type == 'custom') {
                $linkUrl = $item->url;
            } elseif ($item->type == 'system') {
                $linkUrl = $item->reference_id == 'root' ? route('root') : route($item->reference_id);
            } elseif ($item->type == 'page') {
                $page = \App\Models\Page::find($item->reference_id);
                $linkUrl = $page ? route('page', $page->slug) : '#';
            } elseif ($item->type == 'category') {
                $cat = \App\Models\Category::find($item->reference_id);
                $linkUrl = $cat ? route('categoryProducts', $cat->slug) : '#';
            }
        @endphp
        <a href="{{ $linkUrl }}">{{ $item->title }}</a>
    @endforeach
@endif

Seeding

The default menus and their initial structure are populated via database/seeders/MenuSeeder.php. This ensures that when a new user installs the project, the header and footer aren't empty.
